interface CATIUserExitLoginTracker

Interface to track user login attempt.
Role: This interface is a user exit interface. The methods from this interface will be triggered during the login process on failed/successful attempts
You should implement it on the CATUserExitLoginTracker component by creating a data extension.

o UEBuildLoginMessage
public virtual UEBuildLoginMessage( const iUserID,
const iUserRole,
err_code,
oOutMessage)
User exit method to allow building a message to be displayed to the user on a failed/successful login attempt
Role : This method will be triggered at the end of the entire login process when the role requested by the user is applied. This method will be triggered in both cases of success and failure. The purpose of the method is to allow the user to build a custom message that will be displayed on the client at the end of the login process.
Parameters:
iUserID
the user id used for login
iUserRole
the P&O role to be applied for the user This input parameter will be an empty string in case the method is triggered on failure while trying to fetch the applicable roles for the user
err_code
error code to indicate the type of failure in the login process (
ENOVLoginError) When the method is triggered on a successful login attempt, this input value will be ENOVLoginError_NoError. In case of a failed login attempt, this input value will indicate the relevant reason
oOutMessage
Message string that needs to be displayed at the end of the login attempt

public virtual UETrackUserLogin( const iUserID,

User exit method to track the login attempt of the user while fetching and applying roles
Role : This method will be triggered in two cases - 1.) When there are no roles available for the user trying to login (failed attempt to get roles applicable for this user) 2.) At the end of applying the role (successful or failed attempt) being requested for the user login
Parameters:
iUserID
the user id used for login

err_code
error code to indicate the type of failure in the login process (
ENOVLoginError) When the method is triggered on a successful login attempt, this input value will be ENOVLoginError_NoError. In case of a failed login attempt, this input value will indicate the relevant reason

enum ENOVLoginError {
  ENOVLoginError_NoError,
  ENOVLoginError_RootUser,
  ENOVLoginError_UserBlocked,
  ENOVLoginError_UserUnrecognized,
  ENOVLoginError_IncorrectPassword,
  ENOVLoginError_PasswordExpired,
  ENOVLoginError_FileSystemFull,
  ENOVLoginError_NotPnOUser,
  ENOVLoginError_NoPnORole,
  ENOVLoginError_NoLoginPrivilege
}
Additionnal information to indicate the type of error during the ENOVIA login process.
Role: This will be used as an input parameter in , and methods.
Parameters:
ENOVLoginError_NoError
No error during the login process
ENOVLoginError_RootUser
Login attempt is with root user
ENOVLoginError_UserBlocked
User is blocked from login because of number of invalid attempts
ENOVLoginError_UserUnrecognized
User is not a system recognized user id
ENOVLoginError_IncorrectPassword
Input password is incorrect
ENOVLoginError_PasswordExpired
Password is an expired one and login is not allowed
ENOVLoginError_FileSystemFull
File system is full and VDO0OrbServer process can't be launched
ENOVLoginError_NotPnOUser
User is not defined in the P&O
ENOVLoginError_NoPnORole
User does not have any role assigned in the P&O
ENOVLoginError_NoLoginPrivilege
User does not have any role which has the login to ENOVIA privilege

Java Unit Test Manager

Getting Started

How to write and run a test program
Technical Article

Abstract

This task shows you how to get started with Java Unit Test Manager, using a basic test scenario.

  • Populating the Workspace
  • Creating the Test Framework
  • Creating the Testing Code
  • Creating a Test Object
  • Creating JavaLauncher.bat
  • Creating JavaLauncher.sh
  • Building the Workspace
  • Running the Test Object
Before you begin: Note that:
  • You should have first installed the Native Apps Developement Environment.
  • You should also have installed a Dassault Systèmes brand, such as CATIA or DELMIA, as well as the associated API CD ROM. The scenario uses some of the installed frameworks to populate the workspace and run the tools.
  • You should have already created a workspace, such as the one used as an example in Getting Started in C++ Interactive Dashboard (CID). Otherwise, create a simple folder that will be used as the workspace root folder.

Populating the Workspace

Warning: The workspace is supposed to be located in E:\MyWorkspace in this example.

  1. In you workspace, create a folder named CAAApp

  2. In CAAApp, create a folder named IdentityCard.

  3. In the IdentityCard folder, create an XML file named IdentityCard.xml with the following contents:

    <?xml version='1.0' ?>
<testFramework xmlns="http://www.3ds.ic"
               xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
               xsi:schemaLocation="http://www.3ds.ic ICModel.xsd">
</testFramework>

  4. In CAAApp, create a folder named MyJavaModule.mj for the Java module.

  5. In the MyJavaModule.mj module, create the Imakefile.mk file with the following contents:

    BUILT_OBJECT_TYPE=JAVA
LINK_WITH =
  6. In the MyJavaModule.mj module, create the folders src\com\dassault_systemes\myjavacode.
  7. In the myjavacode folder, create the MyJavaCodeClass.java file with the following contents: 
    package com.dassault_systemes.myjavacode;

public class MyJavaCodeClass
{
	public MyJavaCodeClass()
	{	
		System.out.println("MyJavaCodeClass constructor...");
	}
}

The Java class of this framework will be used as the code to test.

Creating the Test Framework

  1. In you workspace, create a folder named CAATest.tst

  2. In CAATest.tst, create a folder named IdentityCard.

  3. In the IdentityCard folder, create an XML file named IdentityCard.xml with the following contents:

    <?xml version='1.0' ?>
<testFramework xmlns="http://www.3ds.ic"
               xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
               xsi:schemaLocation="http://www.3ds.ic ICModel.xsd">
  <prerequisite name = "CAAApp" access = "Public"/>
</testFramework>

  4. In CAATest.tst, create a folder named FunctionTests.

  5. In the FunctionTests folder, create a folder named TestCases.

Creating the Testing Code

  1. In CAATest.tst, create a folder named MyJavaTestModule.mj for the Java module.

  2. In the MyJavaTestModule.mj module, create the Imakefile.mk file with the following contents:

    BUILT_OBJECT_TYPE=JAVA
LINK_WITH = MyJavaModule
  3. In the MyJavaModule.mj module, create the folders src\com\dassault_systemes\myjavatest.
  4. In the myjavatest folder, create the MyJavaTestClass.java file with the following contents: 
    package com.dassault_systemes.myjavatest;

import com.dassault_systemes.myjavacode.MyJavaCodeClass;

public class MyJavaTestClass
{
  public MyJavaTestClass()
  { }

  private int runTest() throws Exception
  {
    MyJavaCodeClass aJavaCodeClass = new MyJavaCodeClass();
    return 0;
  }

  public static void main(String[] args)
  {
    System.out.println("Starting MyJavaTestClass");
    try {
      int rc = new MyJavaTestClass().runTest();
      if (rc!=0) {
        System.exit(rc);
      }
    }
    catch (Throwable e) {
      e.printStackTrace();
      System.exit(777);
    }
  }
}

Creating a Test Object

  1. In the TestCases folder, create an XML file named MyODT.xml with the following contents:

    <?xml version="1.0"?>
<ODT Version="1.0">
  ...
</ODT>

  2. Add the test object definition section.

    <?xml version="1.0"?>
<ODT Version="1.0">
  <Definition>
     <Target desc="This test validates MyJavaCodeClass"/> 
     <MaxTime value="10"/>
  </Definition>
  ...
</ODT>

    The definition section is required and is enclosed within the tag <Definition>. In this example, it limits its replaying duration to 10 seconds.

  3. Add the test object execution section.

    <?xml version="1.0"?>
<ODT Version="1.0"> 
  <Definition> 
    <Target desc="This test validates function xxx"/> 
    <MaxTime value="10"/> 
  </Definition> 
  <Execution> 
    <Command value="CMD /C JavaLauncher.bat" osConstraint="Windows_NT" /> 
    <Command value="/bin/ksh JavaLauncher.sh" osConstraint="Unix" /> 
  </Execution> 
</ODT>

    The execution section is required and is enclosed within the tag <Execution>. In this example, it requests:

    • The JavaLauncher.bat procedure to be run with Windows
    • The JavaLauncher.sh procedure to be run with UNIX.

  4. Save the file MyODT.xml.

Creating JavaLauncher.bat

  1. In CAATest.tst, create the CNext.specifics_Windows_NT/code/command folders.

  2. In the command folder, create a file named JavaLauncher.bat with the following contents:

    @echo off

for /F "delims=" %%i in ( 'WhichDir %OS%\docs\java\MyJavaTestModule.jar' ) do set CLASSPATH=%%i;%CLASSPATH%
for /F "delims=" %%i in ( 'WhichDir %OS%\docs\java\MyJavaModule.jar' ) do set CLASSPATH=%%i;%CLASSPATH%
if not defined JAVA_HOME set JAVA_HOME=%JavaROOT_PATH%
%JAVA_HOME%\bin\java com.dassault_systemes.myjavatest.MyJavaTestClass args1 args2

  3. In the command folder, create a file named WhichDir.bat with the following contents:

    @echo off
setlocal

echo %1 | findstr "^%OS%" > NUL
if %ERRORLEVEL% NEQ 0 (
  for %%I in (%1) do (
    if exist "%%~$ADL_ODT_CONCATENATION:I" (
      echo %%~$ADL_ODT_CONCATENATION:I
      exit /b 0
    )
  )
) else (
  for %%I in (%1) do (
    if exist "%%~$MKMK_LST_RTV:I" (
      echo %%~$MKMK_LST_RTV:I
      exit /b 0
    )
  )
)
echo %1_Not_Found
exit /b 1

Creating JavaLauncher.sh

  1. In CAATest.tst, create the CNext.specifics_UNIX/code/command folders.

  2. In the command folder, create a file named JavaLauncher.sh with the following contents:

    #/bin/ksh

WhichDir()
{
  printf "%s" "$*"| grep "^$OS" > $ADL_ODT_NULL 2>&1
  if [ $? != 0 ];then
    for WSPREQ in $ADL_ODT_CONCATENATION
    do
      if [ "$WSPREQ" != "" ]
      then
        if [ -d $WSPREQ/$1 -o -r $WSPREQ/$1 ]
        then
          echo $WSPREQ/$*
          return 0
        fi
      fi
    done
    printf "%s\n" "${1}_Not_Found"
    return 1
  else
    IFS=":"
    for WSPREQ in $MKMK_LST_RTV
    do
      if [ "$WSPREQ" != "" ]
      then
        if [ -d $WSPREQ/$1 -o -r $WSPREQ/$1 ]
        then
          unset IFS
          echo $WSPREQ/$*
          return 0
        fi
      fi
    done
    unset IFS
    printf "%s\n" "${1}_Not_Found"
    return 1
  fi
}

export CLASSPATH=$(WhichDir $OS/docs/java/MyJavaModule.jar):$CLASSPATH
export CLASSPATH=$(WhichDir $OS/docs/java/MyJavaTestModule.jar):$CLASSPATH
if [ -z "$JAVA_HOME" ];then
                export JAVA_HOME="$JavaROOT_PATH"
fi
$JAVA_HOME/bin/java com.dassault_systemes.myjavatest.MyJavaTestClass args1 args2

Building the Workspace

Warning: The Dassault Systemes 3DEXPERIENCE run time and API are supposed to be installed in C:\Program Files\Dassault Systemes\B27 and while the Development Tools can be installed in the same place, it is recommended to install them in another localtion like in C:\Program Files\Dassault Systemes\T27. Replace these paths with your own Dassault Systemes V5 installation folder and the Development Tools installation folder respectively if you installed them in other places.

  1. Open a command prompt.

  2. Set the current folder at the root folder of your workspace.

  3. Type:

    C:\Program Files\Dassault Systemes\T27\win_b64\code\command\tck_init.bat

    This command initializes the tools to be used to build the workspace.

  4. Type:

    tck_list

    This command lists the available TCKs. Choose a value similar to V5_6A2017_1524_B27.

  5. Type:

    tck_profile V5_6A2017_1524_B27

    The build environment is now set up.

  6. Type:

    mkGetPreq -p "C:\Program Files\Dassault Systemes\B27"

    The prerequisites of the frameworks in your workspace are now set up.

  7. Type:

    mkmk -au

    The frameworks are now built.

  8. Type:

    mkCreateRunTimeView

    The run time view of your workspace is created.

Running the Test Object

  1. Still in the command prompt set at the root folder of your workspace, type:

    mkodt -s MyODT

    Note that the test object name if the name of the XML file without its extension.

The test object output is:

## ------------------------------------------------------------
## Workspace Directory : E:\MyWorkspace
## Concatenation : E:\MyWorkspace 
## Concatenation : C:\Program Files\Dassault Systemes\B27
## OS : win_b64
## Machine : LW5PSRDSY
## Display :
## 1 ODT to replay
## ------------------------------------------------------------
## RUNNING ODT [MyFW.tst MyODT] (1 / 1)
max_time: 10 min.
elapsed_time: 00 min 01 sec
## ++ MyODT : Test execution successful code 0
## ------------------------------------------------------------
##
## 1 ODT replayed, 0 KO.

The test object traces are located in E:\MyWorkspace\CAATest.tst\FunctionTests\Output\win_b64 or aix_a64. The trace file name is the ODT name, MyODT, with extension traces, that is, MyODT.traces. It contains the traces issued by the tested code, as well as possible error messages.

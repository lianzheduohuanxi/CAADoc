Abstract

This article describes the problems you may encounter when using RADE tools in Rational Software Architect Designer and the corresponding workarounds.

  • CAA Plugins are not Displayed or Do Not Work Properly
  • A New Version of CAA Plugins Cannot Be Installed

CAA Plugins are not Displayed or Do Not Work Properly

Sometimes, when RADE tools are installed on top of Rational Software Architect Designer IDE, CAA plugins are not displayed. To enable CAA plug-ins:
  • Create a temporary directory.
  • Move all CAA plug-ins installed for CAA-RADE INSTALL_RADE\intel_a\resources\CATVBTServer\CAAExtensionSiteRSx\eclipse\plugins\com.dassault_systemes.* to this temporary directory.
  • Launch Rational Software Architect Designer with option -clean.
  • Launch Rational Software Architect Designer IDE. CAA plugins are not referenced any longer.
  • Move back CAA plugins from the temporary directory to INSTALL_RADE\intel_a\resources\CATVBTServer\CAAExtensionSiteRSx\eclipse\plugins.
  • Launch Rational IDE. CAA plugins are still not displayed.
  • Exit Rational IDE and relaunch a second time. CAA plug-ins are displayed.
Another cleanup can be done also:
  • Go in directory C:/Users/Your_Username/.eclipse, and rename directory org.eclipse.platform_xxx to sav.org.eclipse.platform_xxx
  • Start Rational Software Architect Designer. Restart could take a little longer since this directory will be recreated from scratch

A New Version of CAA Plugins Cannot Be Installed

if you cannot install the DMC plugin, with following error message :
The items selected for install include a software feature (com.ibm.rcp.dombrowser.feature.feature.jar) that requires install procedures from the older Eclipse Update Manager. The installation cannot continue.

  • This problem is due to an IBM Rational Software 9.0 issue. Please refer to this article for more information and how to solve the problem.
    http://www-01.ibm.com/support/docview.wss?uid=swg21640442

For any other error, please read the following :

Sometimes, when trying to install a new version of RADE plugins in Rational Software Architect Designer, an error message may occurs in the installation wizard and the plugin cannot be installed. In order to solve this problem you can try to clean the .eclipse directory as described before, or you can also try to launch Rational Software Architect Designer with the -clean option. In order to do that, proceed as follow:
  • Select Rational Software Architect Designer icon on your desktop or in the Start menu, and click "Properties"
  • Copy the "Target" field value
  • Open a "cmd" window
  • Paste the value in the cmd window and add the option "-clean", press enter
  • Once Rational Software Architect Designer is launched, retry to install RADE plugins using "Help/Install New Software" menu
Warning: It is important to understand some mecanisms for plugins installation in Rational Software Architect Designer:
Depending on the way Rational Software Architect Designer is launched (“Run as Administrator” or as a simple user), there could be an impact from installation and runtime perspectives:
  • If a plugin is installed as Administrator, then the plugin directory is copied in the Rational Software Architect Designer Installation (ie: C:\Program Files\IBM\SDP\plugins). It means that the installed plugin will be taken into account at runtime for all users.
  • If a plugin is installed as a simple user, then the plugin directory is copied in a .eclipse directory for that user (i.e.: C:\Users\theUsername\.eclipse) and are not copied in the in the Rational Software Architect Designer Installation. It means that the installed plugin will be taken into account at runtime only for that user.

As a consequence, the following behavior happens:

  • As a simple user, if you encounter an issue when trying to update a new version of a plugin previously installed by yourself (in the C:\Users\theUsername\.eclipse directory) then you can try to rename the .eclipse directory prior to run the plugin installation. It should allow installing the new plugin version with no error. Of course, be careful if several plugins were installed in this .eclipse directory.

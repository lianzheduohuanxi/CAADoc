Opening a Workspace

This task shows you how to open a workspace.

Before you begin: Note that:
  • You can open a workspace from the File menu or from the 3DS Workspace Explorer.
  • You can also open a recently opened workspace from the Visual Studio start page.
  • You can disable run time computing.
  • By default, the Visual Studio solution associated with the workspace is created in the ToolsData\VisualStudio2015 folder below the workspace root folder. You can choose to manage it in another folder.
Warning: If you work with several V5 and V6 release levels, always check that the tools initialization command path is set to the tools installation path of the release level used by the workspace to open. For example, when you want to open a V5R28 workspace, set the tools initialization command path to the one in your V5R28 CAA RADE installation path. When afterwards you want to open a V6R2018x workspace, reset the tools initialization command path to the one in your V6R2018x Native Apps Development Toolset installation path. See Accessing and Customizing Settings.

  1. Open the 3DS - Open Workspace dialog box using one of these ways:

    • On the File menu, point to Open and click 3DS Workspace.
    • In the 3DS Workspace Explorer, click .

    If a workspace is current and displayed in the 3DS Workspace Explorer, a prompt asks you whether you want to save its associated solution.

    If the 3DS Workspace Explorer was not opened, it opens, and the Open Workspace dialog box opens.

  2. In Location, either click Browse to select the root folder of an existing workspace, or select one among the last opened workspaces listed.

  3. In Tool level, select the tools level you want to use for this workspace.

  4. To disable run time environment computing, click in front of Open Options and check Disable run time environment computing.

    Disabling run time environment computing improves workspace opening performance if you do not need the run time environment during your working session, that is, for example if you do not need to run tests or debug your code.

  5. To create the workspace associated solution in another folder than the default one, in Associated Visual Studio solution location, either click Browse to select, or type the path of an existing solution.

  6. Click Open.

    The workspace is opened. The 3DS Workspace Explorer is refreshed with the opened workspace. If the Solution Explorer is displayed, it is refreshed with the solution associated with the workspace.

Note: You can remove workspaces from the Location list using the Edit  open list history command in the setting section. See Accessing and Customizing Settings.

Generating an ENOVIA Plugin  Module

This task shows you how to generate an ENOVIA plugin module. Such a module is required to afterwards generate either an ENOVIA event listener or an ENVIA user exit.
Before you begin:
  • Open a workspace.
  • This workspace must contain a framework with the following frameworks as prerequisites with Public access: System. See Creating a Framework, and Editing Framework Identity Card.
  • Display the Solution Explorer.

  1. In the Solution Explorer, right click the project named with the framework name, that is, NOT the node prefixed with FW_, and containing the framework to which you want to add a plugin module.

  2. In the contextual menu, point to Add, and click New Item.

    The Add New Item dialog box opens.

  3. In Installed, click the arrow in front of 3DS Templates to expand its contents, if not yet expanded.

  4. Click ENOVIAV5.

    The ENOVIAV5 installed templates are displayed in the central pane.

  5. In the central pane, click CATPlugIn.

  6. In the Name box, type the name you want to assign to the plugin module.

    Warning: This name is used to create the test file name. Bear in mind that:

    • All the characters are allowed except those listed below:

      * ? \ / < > " ' | :

    • With Windows, full path names must not exceed 260 characters.

  7. Click Add.

    The plugin is created.

    Warning: If you do not see all the created folders and files, in the 3DS Workspace Explorer, right click the framework name, and click Refresh Solution.

    Assume that the plugin name is NewPlugin. The framework includes now the following:

    Folders and Files in the Solution Explorer Description
    In CNext/code/dictionary
    File Description To Customize
    CAAPlugInFW.dico The framework dictionary file. No
    In CNext/resources/msgcatalog
    File Description To Customize
    PlugInList.CATRsc The plugin resource file. Yes
    In IdentityCard
    File Description To Customize
    IdentityCard.xml The framework's identity card file is not updated. You will update it below. You will need to update also it when creating an ENOVIA event listener or a user exit. Yes
    In ProtectedIInterfaces
    File Description To Customize
    NewPlugin.h The module header file. No
    In NewPlugin.m/LocalInterfaces
    File Description To Customize
    ENOVPluginNewPlugin.h The plugin class header file. No
    ENOVPluginNewPluginListener.h The event listener class header file. Yes
    In CAAAfrGeoCreationWkbMod.m/src
    File Description To Customize
    ENOVPluginNewPlugin.cpp The plugin class file. No
    ENOVPluginNewPluginListener.cpp The event listener class file. Yes
    In NewPlugin.m
    File Description To Customize
    Imakefile.mk The plugin module Imakefile. No

  8. Open the IdentityCard.xml file and add the VPMInterfaces framework as a public prerequisite.

  9. You can now generate in this plugin module:

    • An ENOVIA event listener: see Generating an ENOVIA Event Listener.
    • An ENOVIA user exit: see Generating an ENOVIA User Exit.

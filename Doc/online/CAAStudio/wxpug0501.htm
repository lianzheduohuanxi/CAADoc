Managing Prerequisites

The prerequisite frameworks of your workspace are the frameworks from which you use classes, interfaces, or global functions.

  • Adding a Prerequisite Workspace
  • Removing a Prerequisite Workspace
  • Changing Prerequisite Concatenation Order
  • Declaring the Start Program
Before you begin:
  • Open a workspace with at least a framework and a module.
  • Update the framework IdentityCard file.

Adding a Prerequisite Workspace

  1. In the 3DS Workspace Explorer, right click the workspace name, and click Prerequisites.

    The 3DS - Manage prerequisites dialog box opens.

  2. For each prerequisite workspace to declare:

    1. Click .

      The Select Folder dialog box opens.

    2. Locate a prerequisite workspace root folder and select it.

      At least you should add the folder where you have installed the brand CD ROM (usually C:\Program Files\Dassault Systemes\B28).

    3. Click Select Folder.

      The Select Folder dialog box closes and the selected workspace folder is displayed and selected in the top box.

  3. If the exe displayed in Start program for dll that comes from the one set in the Options dialog box (see Accessing and Customizing Settings) is not convenient, replace it using the one you wish for your workspace. This exe is the one run to start your applications.

  4. If you want to copy the frameworks from this (these) prerequisite workspace(s) to a local folder:

    1. Check Make a copy of the prerequisite files.

    2. In Target directory, click Browse to select a folder where the frameworks will be copied (or type the path name of this folder).

      If you have clicked Browse, the Select Folder dialog box opens.

    3. Locate the a folder where the frameworks will be copied, and select it.

    4. Click Select Folder.

      The Select Folder dialog box closes and the selected folder is displayed in the Target directory box.

  5. Click OK.

    The declared workspace(s) is (are) added to your workspace prerequisites, possibly copied to the folder you have selected, and the identity cards of your frameworks are rebuilt, taking this (these) workspace(s) into account. The start program defined in Start program for dll is also copied to your workspace from the first workspace in which it is found in the order they appear in the dialog box. See Accessing and Customizing Settings.

    Each class, interface, or global function you use from these prerequisite workspaces are retrieved from the first workspace in which it is found in the order these workspaces appear in the dialog box.

Removing a Prerequisite Workspace

  1. In the 3DS Workspace Explorer, right click the workspace name, and click Prerequisites.

    The 3DS - Manage prerequisites dialog box opens.

  2. Click the root folder of the workspace to remove from your workspace prerequisites.

  3. Click .

    The workspace is removed from your workspace prerequisites.

  4. Click OK to confirm the workspace removal from the prerequisites.

Changing Prerequisite Concatenation Order

  1. In the 3DS Workspace Explorer, right click the workspace name, and click Prerequisites.

    The 3DS - Manage prerequisites dialog box opens.

  2. Click the root folder of the workspace to move.

  3. Click to move it upwards, or to move it downwards.

  4. Click OK to confirm the workspace prerequisite concatenation order change.

Declaring the Start Program

  1. In the 3DS Workspace Explorer, right click the workspace name, and click Prerequisites.

    The 3DS - Manage prerequisites dialog box opens.

  2. In Start program for dll, type the name of the program to run, usually CNext.exe.

    This start program must be an exe. It is copied from the fist prerequisite workspace in which it is found in the concatenation to the win_b64\code\bin folder of your workspace. It is used when debugging a DLL and set as the program to run when you ask for debugging a DLL in the Solution Explorer.

    Warning: When you set an exe in this dialog box, it is assigned to all the DLLs you will project afterwards to the Solution Explorer, but not for those already projected, unless you run a refresh, either globally at the workspace level, or locally at the framework or module level containing these DLLs projected before this exe was set.

  3. Click OK to confirm this start program copy.

Generating an ENOVIA Event Listener

This task shows you how to generate an ENOVIA event listener.
Before you begin:
  • Open a workspace.
  • This workspace must contain a framework with the following frameworks as prerequisites with Public access: System and VPMInterfaces. See Creating a Framework, and Editing Framework Identity Card.
  • A plugin module must exist in this framework. See Generating an ENOVIA Plugin Module.
  • Display the Solution Explorer.

  1. In the Solution Explorer, right click the project named with the plugin module name to which you want to add an ENOVIA event listener.

  2. In the contextual menu, point to Add, and click New Item.

    The Add New Item dialog box opens.

  3. In Installed, click the arrow in front of 3DS Templates to expand its contents, if not yet expanded.

  4. Click ENOVIAV5.

    The ENOVIAV5 installed templates are displayed in the central pane.

  5. In the central pane, click CATEventListener.

  6. In the Name box, type the name you want to assign to the event listener class.

    Warning: This name is used to create the test file name. Bear in mind that:

    • All the characters are allowed except those listed below:

      * ? \ / < > " ' | :

    • With Windows, full path names must not exceed 260 characters.

  7. Click Add.

    The 3DS - ENOVIA Event Listener opens.

     

  8. In the Object Name box, possibly change the name of the ENOVIA event listener.

  9. In the Domain list, select the domain to which you want to generate an ENOVIA event listener.

  10. In the Class Name list, select the class you want to listen to.

  11. In the Event to subscribe to list, select the event you want to listen to.

  12. Click Finish.

    The ENVIA event listener is generated.

    Warning: If you do not see all the created folders and files, in the 3DS Workspace Explorer, right click the framework name, and click Refresh Solution.

    Assume that the event listener name is EvtListener1. The framework includes now the following:

You can now implement the ENOVIA event listener.

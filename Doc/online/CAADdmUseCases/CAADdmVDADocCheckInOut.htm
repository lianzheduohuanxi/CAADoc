Check In and Check Out a document(s) from ENOVIA vault

What You Will Learn With This Use Case

This use case is intended to show you how to Check In and Check Out a document(s) from ENOVIA vault or is controlled by the system.

Documents, whether they are internal (the actual document resides in an ENOVIA vault and is controlled by the system) or external (paper or residing in a system outside of ENOVIA) may be checked in or out.

This allows for both version control and enterprise sharing. Checking In a document creates a link between the document record information and a physical document file. Check Out procedure, which allows you to update the document on your own computer.

[Top]

The CAADdmVDADocCheckInOut Use Case

CAADdmVDADocCheckInOut is a use case of the CAAENOVDesktopDocument.edu framework that illustrates document(s) Check In and Check Out capabilities from ENOVIA vault.

[Top]

What Does CAADdmVDADocCheckInOut Do

CAADdmVDADocCheckInOut use case:

  • Creates a new login session in the current application
  • Create a Document Master and Revision
  • Create ENOVIAVDADocCheckIn Interface
  • Setting object info before execute CheckIn command
  • Get ENOVIA_CHECKIN_PATH to Check In the file
  • Execute CheckIn command of ENOVIAVDADocCheckIn Interface
  • Then it proceed for check out, Create ENOVIAVDADocCheckOut interface
  • Setting object info before execute CheckOut command
  • Get ENOVIA_CHECKOUT_PATH to Check Out the file
  • Execute CheckOut command of ENOVIAVDADocCheckOut interface
  • Later Retrieve Vault info from CheckOut command
  • Finally, close the session

How to Launch CAADdmVDADocCheckInOut

To launch CAADdmVDADocCheckInOut, you will need to:

  • Set up the build time environment, then compile CAADdmVDADocCheckInOut along with its prerequisites.
  • Set up the run time environment.
  • Then set the ENOVIA_CHECKIN_PATH and ENOVIA_CHECKOUT_PATH environment variables with some local directory path (Example: set ENOVIA_CHECKIN_PATH=C:\temp and set ENOVIA_CHECKOUT_PATH=C:\temp\CheckOut) before launch of this use case in the same command prompt window. Please note that there should be no space at the beginning of the path that is being set for these variables.
  • Make sure that there is a SampleDoc.txt file in the directory where ENOVIA_CHECKIN_PATH path has been defined. In our example have a SampleDoc.txt file in C:\temp
  • Launch the use case by executing the following command:

    • mkrun -c "CAADdmVDADocCheckInOut"

Create a New Login Session

CATIVpmFactoryManager_var spTheFactory;
If (piSess_handle != NULL)
    rc = piSess_handle->GetVPMObjectFactory (spTheFactory);
CATIVpmSCLogin_var spTheLogin (spTheFactory);
CATUnicodeString user;
CATUnicodeString pwd;
CATUnicodeString role;
VPMLoginPolicy iRoleChoicePolicy = VPMLoginDEFAULT;
rc = spTheLogin->Login (user, pwd, iRoleChoicePolicy, role);
if (FAILED (rc))
{
   cout << "login pb" << endl << flush;
   VPMSession::CloseSession ();
   return 101;
}    
cout << "Login session successfully created" << endl << flush;

In order to create a new login session, use the login method of CATIVpmSCLogin. This method is used to get the login session corresponding to a specific user. The method return HRESULT. Once HRESULT is SUCEEDED then it is assumed that the user has been authenticated.

Create an ENOVIAVDADocCheckIn Command

CATIAVPMVDACommand *piCommand = NULL;
CATIAVPMVDACommandFactory * piFactory = GetDocCommandFactory();
rc = piFactory->Create( "VDADocCheckIn", piCommand );  
if ( !SUCCEEDED(rc) || !piCommand )
     return 6;
cout << "CheckIn Command has been created" << endl << flush;

To Create an ENOVIAVDADocCheckIn command, first create a CATIAVPMVDACommandFactory pointer then call Create method through CATIAVPMVDACommandFactory pointer, this Create method of CATIAVPMVDACommandFactory return a pointer of a type CATIAVPMVDACommand. ENOVIAVDADocCheckIn interface checks in a given document revision with/without a file (physical file, vault file, or external file, etc.). It can also be used to undo the last check-out

In Short

This use case has illustrated how to Check In and Check Out a document(s) from ENOVIA vault or is controlled by the system. Initially, the login session is created using CATIVpmSCLogin::Login method. Then obtain DDManager using CATIEnovDDManager::GetRootDesktop method. Then create a document using CATIEnovDDManager::CreateDocument method. The create ENOVIAVDADocCheckIn interface using CATIAVPMVDACommandFactory::Create method. Now, set the object info before execute CheckIn command using ENOVIAVDADocCheckIn::put_BObject method. Then Execute Check In command. To Create ENOVIAVDADocCheckOut interface use CATIAVPMVDACommandFactory::Create method. To set object info before execute CheckOut command using ENOVIAVDADocCheckOut::put_BObject method. Then Execute Check Out command. Finally close the session.

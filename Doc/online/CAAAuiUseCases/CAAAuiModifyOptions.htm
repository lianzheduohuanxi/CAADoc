What You Will Learn With This Use Case

This use case is intended to help you make your first steps in programming with CATIA Assembly Design. Its main intent is change general and constraint related options. More specifically, you will learn how to:

  • Search for settings controller for Assembly Design General options.
  • Modify general options.
  • Search for settings controller for Assembly Design Constraints options.
  • Modify constraints options.

[Top]

The CAAAuiModifyOptions Use Case

CAAAuiModifyOptions is a use case of the CAAAssemblyUI.edu framework that illustrates the CATAssemblyInterfaces framework capabilities.

Before discussing the use case, some main concepts have to be introduced:

  • Manual Update Mode :- Update command will have to be launched manually.

  • Automatic Update Mode :- Update command will have to be launched automatically.

  • Extend Move To FixTogether :- All components involved in a FixTogether will be moved together during manipulation.

  • Paste Component With Constraints On Copy :- Copied component's constraints only be recreated after a Copy.

  • Paste Component With Constraints On Cut :- Copied component's constraints will only be recreated after a Cut.

  • Paste Component With Constraints On Copy and Cut :- Copied component's constraints will be recreated after a Copy or a Cut.

  • Paste Component Without Constraints:- Copied component's constraints will not be recreated.

[Top]

What Does CAAAuiModifyOptions Do ?

CAAAuiModifyOptions does the following:

  • Searches for settings controller for Assembly Design General options.
  • Modify general options.
  • Search for settings controller for Assembly Design Constraints options.
  • Modify constraints options.

[Top]

How to Launch CAAAuiModifyOptions

To launch CAAAuiModifyOptions, you will need to set up the build time environment, then compile CAAAuiModifyOptions along with its prerequisites, set up the run time environment, and then execute the use case [1].

Launch the use case as follows:

  • With Windows 
    e:>CAAAuiModifyOptions
  • With UNIX 
    $ CAAAuiModifyOptions

[Top]

Where to Find the CAAAuiModifyOptions Code

The CAAAuiModifyOptions use case is located in the CAAAuiModifyOptions.m module of the CAAAssemblyUI.edu framework:

Windows InstallRootDirectory\CAAAssemblyUI.edu\CAAAuiModifyOptions.m\
Unix InstallRootDirectory/CAAAssemblyUI.edu/CAAAuiModifyOptions.m/

where InstallRootDirectory is the directory where the CAA CD-ROM is installed.

Step-by-Step

There are eight main steps in CAAAuiModifyOptions:

  1. Prolog
  2. Getting settings controller for Assembly Design General options
  3. Setting automatic update mode
  4. Setting Extend Move To FixTogether option
  5. Getting settings controller for Assembly Design Constraints options
  6. Setting Paste Component With Constraints option
  7. Setting ordered types list for quick constraint option
  8. Epilog

We will now comment each of those sections by looking the code.

[Top]

Prolog

CAAAuiModifyOptions begins by creating a session.

Getting settings controller for Assembly Design General options

  ...
      CATIAsmGeneralSettingAtt * piGnrlSettingAtt = NULL;
      HRESULT rc = GetCATAsmGeneralSettingCtrl( IID_CATIAsmGeneralSettingAtt, (void **) & piGnrlSettingAtt );
      // If rc succeeds, settings controller is retrieved.
  ...

[Top]

Setting automatic update mode

  ...
      rc = piGnrlSettingAtt->SetAutoUpdateMode( CATIAsmGeneralSettingAtt::AutomaticUpdate );
      // If rc succeeds, update mode is set to automatic.
  ...

[Top]

Setting Extend Move To FixTogether option

  ...
      rc = piGnrlSettingAtt->SetMoveWithFixTExtendMode( CATIAsmGeneralSettingAtt::AlwaysExtendMove );
      // If rc succeeds, all components involved in a FixTogether will be moved.
  ...

[Top]

Getting settings controller for Assembly Design Constraints options

  ...
      CATIAsmConstraintSettingAtt * piCstSettingAtt = NULL;
      rc = GetCATAsmConstraintSettingCtrl( IID_CATIAsmConstraintSettingAtt, (void **) & piCstSettingAtt );
      // If rc succeeds, settings controller is retrieved.
  ...

[Top]

Setting Paste Component With Constraints option

  ...
      rc = piCstSettingAtt->SetPasteComponentMode( CATIAsmConstraintSettingAtt::WithCstOnCut );
      // If rc succeeds, copied component's constraints will be recreated after a Cut.
  ...

[Top]

Setting ordered types list for quick constraint option

  ...
      char ** typesList = NULL;
      int size = 0;
      rc = piCstSettingAtt->GetQuickConstraintOrderedList( typesList, size );
      if ( SUCCEEDED( rc ) && NULL != typesList )
      {
        if ( 2 <= size )
        {
          char * tmp = typesList[0];
          typesList[0] = typesList[1];
          typesList[1] = tmp;
          rc = piCstSettingAtt->SetQuickConstraintOrderedList( (const char **) typesList, size );
        }
      }
      // If rc succeeds, ordered types list for quick constraint option will be set.
  ...

Epilog

  ...
      Delete_Session("Session_ASSEMBLY");
  ...

What You Will Learn With This Use Case

This use case is intended to help you make your first steps in programming with CATIA Assembly Design. Its main intent is to refresh constraints of a product. More specifically, you will learn how to:

  • Load documents in session. The names of the documents to load are parameters of the command.
  • Insert a part under a product.
  • Refresh constraints of the product.

[Top]

The CAAAuiRefreshCsts Use Case

CAAAuiRefreshCsts is a use case of the CAAAssemblyUI.edu framework that illustrates the CATAssemblyInterfaces framework capabilities.

[Top]

What Does CAAAuiRefreshCsts Do ?

CAAAuiRefreshCsts does the following:

  • Opens a CATProduct document.
  • Inserts an existing component under the root product: the part named RefreshCstPart1.
  • Sets the name of the inserted instance as "Part1.1".
  • Refreshes constraints under the root product.

[Top]

How to Launch CAAAuiRefreshCsts

To launch CAAAuiRefreshCsts, you will need to set up the build time environment, then compile CAAAuiRefreshCsts along with its prerequisites, set up the run time environment, and then execute the use case [1].

Launch the use case as follows:

  • With Windows 
    e:>CAAAuiRefreshCsts InputDirectory\Root_RefreshCst.CATProduct InputDirectory\RefreshCstPart1.CATPart
  • With UNIX 
    $ CAAAuiRefreshCsts InputDirectory/Root_RefreshCst.CATProduct InputDirectory/RefreshCstPart1.CATPart

where:

InputDirectory: The directory into which the documents Root_RefreshCst.CATProduct and RefreshCstPart1.CATPart are stored, usually in the resources directory of your RuntimeView

Where to Find the CAAAuiRefreshCsts Code

The CAAAuiRefreshCsts use case is located in the CAAAuiRefreshCsts.m module of the CAAAssemblyUI.edu framework:

Windows InstallRootDirectory\CAAAssemblyUI.edu\CAAAuiRefreshCsts.m\
Unix InstallRootDirectory/CAAAssemblyUI.edu/CAAAuiRefreshCsts.m/

where InstallRootDirectory is the directory where the CAA CD-ROM is installed.

Step-by-Step

There are eight main steps in CAAAuiRefreshCsts:

  1. Prolog
  2. Inserting an existing component under the root product
  3. Renaming the inserted instance
  4. Refreshing constraints of the root product
  5. Epilog

We will now comment each of those sections by looking the code.

[Top]

Prolog

CAAAuiRefreshCsts begins by checking that the command lines contains two arguments along with the command name "CAAAuiRefreshCsts". Then it creates a session and loads documents whose names are passed as arguments. If documents does not exist, code 2 is returned.

Then CAAAuiRefreshCsts retrieves the product document's root product, handled by the pointer piProductOnRoot which is a pointer to CATIProduct interface. It also retrieves the part document's root, handled by the pointer spPrd_PartRoot which is also a pointer to CATIProduct interface. This is the usual sequence to browse a product structure [3] and find the root product pointer to CATIProduct.

piProductOnRoot A pointer to CATIProduct onto the root of product document
spPrd_PartRoot A pointer to CATIProduct onto the root of part document

[Top]

Inserting an existing component under the root product

  ...
      CATIProduct_var spProduct;
      spProduct = piProductOnRoot->AddProduct( spPrd_PartRoot );
      // If spProduct is not NULL, inserting part succeeded.
  ...

[Top]

Renaming the inserted instance

  ...
      spProduct->SetPrdInstanceName( "Part1.1" );
  ...

[Top]

Refreshing constraints of the root product

  ...
      rc = CATAsmConstraintServices::RefreshConstraintsInProduct( piProductOnRoot );
      // If rc succeeds, constraints are refreshed.
  ...

[Top]

Epilog

The document is removed and the session is closed

  ...
      CATDocumentServices::Remove(*pProductDocument);
      CATDocumentServices::Remove(*pPartDocument);
      Delete_Session("Session_ASSEMBLY");
  ...

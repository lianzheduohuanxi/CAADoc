What You Will Learn With This Use Case

The CAAxPDMDisplayPropertiesUse Case

CAAxPDMDisplayPropertiesis a use case of the CAAxPDMInterfaces.edu framework that illustrates CATxPDMInterfaces framework capabilities.
Be careful, such as all use cases illustrating the CATxPDMInterfaces framework, the current one requires a PX1 license.

[Top]

What Does CAAxPDMDisplayPropertiesDo

  • Launch CATIA.
  • Create or load documents, for example following structure:

  • Retrieve the PX1 CAASample Toolbar
    • View / Toolbars / PX1 CAASample
  • Click on 'Connect To PX1' icon to active toolbar commands.

  • Select the Display Item Properties command
  • When you click on the icon, you have this dialog box
    Fig.1 xDPM Properties Dialog Box

    By default, properties of current item are displayed (blue node in Product Structure or current CATPart or CATDrawing).
    You can change current item by:

    • Selecting in 3D or graph another object
    • Double click on one item in "Children" or "Import" tabs
    • Click on "Back to Previous Item" button
  • In first tab, all general information is displayed:
    • Name and path of item (CATIxPDMItem::GetDocFileName). Path could be empty is document has never been saved.
    • Type of item (CATIxPDMItem::GetItemType)
    • Preview (CATIxPDMItem::GeneratePreview)
    • Session Information (Access Rights and Modification Status)
    • File Information (only after first save).
  • In second tab, all properties are displayed:

  • In third tabs, all children are displayed.
    For CATProduct, CATIxPDMProductItem::GetChildren is used. So Instance name are displayed. If you double click on one line, panel switch to child with minimum load of data.
    For CATPart and CATDrawing, CATIxPDMItem::GetChildren is used.

  • Last tab is available only on CATPart item. It displays import links.
    Activation of this tab load all necessary data to compute these links (it could take long time on very big data).
    On can go directly to pointed item by double-clicking on line.

How to Launch CAAxPDMDisplayProperties

To launch CAAxPDMDisplayProperties, you will need to set up the build time environment, then compile CAAxPDMUICommands with its prerequisites, set up the run time environment, and then execute the use case [1]. To launch it, refer to the What Does CAAxPDMDisplayProperties Do section.

[Top]

Where to Find the CAAxPDMDisplayProperties Code

The CAAxPDMDisplayProperties use case is made of three files,

  • The CAAxPDMToolbar class, located in the CAAxPDMToolbar.m module of the CAAxPDMInterfaces.edu framework:
    Windows InstallRootDirectory\CAAxPDMInterfaces.edu\CAAxPDMToolbar.m\
    Unix InstallRootDirectory/CAAxPDMInterfaces.edu/CAAxPDMToolbar.m/

    This file contains the add-in to create the toolbar.

  • The CAAxPDMPropertyCmd class and the CAAxPDMPropertyPanel class
    Windows InstallRootDirectory\CAAxPDMInterfaces.edu\CAAxPDMUICommands.m\
    Unix InstallRootDirectory/CAAxPDMInterfaces.edu/CAAxPDMUICommands.m/

    The first class is the state command launched when the end user push Display Item Properties command. The second one defines the dialog box [Fig.1] launched by the CAAxPDMPropertyCmd command.

where InstallRootDirectory is the directory where the CAA CD-ROM is installed.

# CAA 知识内化 — 源码深度阅读与知识蒸馏

## 原则

- **不依赖脚本批量处理**：每一行代码人工阅读、理解、内化
- **输出可读的知识文档**：每个模块输出一份独立的知识文档，而非机械转换
- **分层递进**：从核心框架到专业模块，按依赖关系阅读

## 项目总览

73 个 .edu 项目，按依赖关系和重要程度分为 7 个层级：

```
Tier 1  核心基础 (5 projects)  ← 当前
Tier 2  UI/对话框 (3 projects)
Tier 3  机械建模核心 (6 projects)
Tier 4  产品结构 (3 projects)
Tier 5  加工制造 (4 projects)
Tier 6  专业领域 (12 projects)
Tier 7  小型/工具 (40 projects)
```

## 执行约定

1. **每次处理一个 Tier 中的一个项目**
2. **阅读流程**:
   - 先读 `PublicInterfaces/*.h` — 理解对外契约
   - 再读 `ProtectedInterfaces/*.h` — 理解内部扩展点
   - 然后读 `LocalInterfaces/*.h` — 理解实现类
   - 最后读 `.cpp` — 理解具体实现逻辑
3. **输出格式**: `knowledge/{tier}/{ProjectName}-{中文名}.md`
   - 包含：模块定位、核心接口列表（含方法说明）、架构图（文字描述）、关键设计模式、代码示例
4. **不使用脚本批量处理** — 每个文件人工阅读

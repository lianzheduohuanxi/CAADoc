# CAA知识库 AI Agent 检查补充方案

## 📋 问题诊断

### 当前蒸馏产物的已知问题

通过抽样检查，发现以下严重问题：

| 问题类别 | 严重程度 | 影响范围 | 说明 |
| HTML→MD代码格式丢失 | 🔴 严重 | 全部1149个文档 | html2text无法保留`<pre>`代码块格式，代码被混入正文 |
| API接口大量遗漏 | 🔴 严重 | 仅提取23/863个 | 正则只匹配有virtual方法的接口，遗漏空接口和声明式接口 |
| 快速参考代码语法错误 | 🔴 严重 | 手写的6个文档 | `CATImplementInterface`语法完全错误（虚构了不存在的宏） |
| HTML表格转换混乱 | 🟡 中等 | 大部分文档 | html2text将HTML表格转为无结构的文本 |
| Frontmatter标签噪音 | 🟡 中等 | 全部文档 | tags中包含大量非API的CAA前缀词（如CAACatiaApplicationFrm） |
| 文档分类混乱 | 🟡 中等 | 输出目录 | 模块名映射为小写后不直观（caaafrcases） |
| QI路径零提取 | 🔴 严重 | 0条 | edu源码中无显式QI实现，需从TIE文件和宏推断 |
| 故障排除内容虚构 | 🔴 严重 | 3个文档 | 编译器/链接器/运行时错误指南为AI生成的通用内容，非CAA特有 |

### 详细问题分析

#### 问题1: HTML→Markdown 代码格式丢失

**原始HTML** (Doc/online/CAAAfrUseCases/CAAAfrCheckHeader.md):
```html
<pre>void CAAAfrGeneralWksAdn::CreateCommands(#)
{
  CATAfrCheckHeaderAccessor <b>ViewerFeedbackHdrAccessor</b> (&quot;CAAAfrViewerFeedbackHdr&quot;);
  ViewerFeedbackHdrAccessor.<b>SetResourceFile</b>(&quot;CAAAfrGeneralWksAddinHeader&quot;);
</pre>
```

**转换后** (丢失代码块标记):
```markdown
void CAAAfrGeneralWksAdn::CreateCommands(#) { CATAfrCheckHeaderAccessor ...
```
→ 代码没有 ` ```cpp ` 围栏，被当作正文处理

#### 问题2: 接口遗漏原因

正则 `class/s+/w+/s+(/w+)/s*:/s*public/s+(/w+)` 匹配条件：
- 必须有 `virtual ... = 0` 方法 → 遗漏空接口（如 `CATICAAAfrGeometryWksConfiguration`）
- 只扫描 PublicInterfaces → 遗漏 LocalInterfaces（751个头文件）
- 不匹配 `CATDeclareInterface` 宏 → 这是CAA真正的接口声明方式

**实际分布：**
- PublicInterfaces CAA*.h: 112个
- LocalInterfaces CAA*.h: 751个
- 当前提取: 仅23个

#### 问题3: 快速参考语法错误

**我们写的（错误）**:
```cpp
CATImplementInterface(ClassName, CATBaseUnknown)
  CATImplementInterfaceMethod(YourMethod)    // ← 不存在的宏
EndCATImplementInterface                      // ← 不存在的宏
```

**实际CAA写法**:
```cpp
CATImplementInterface(CAAIAfrMRUManagement, CATBaseUnknown);  // 一行即可
```

## 🎯 AI Agent 检查补充方案

### 总体策略

```
检查顺序: 高影响 → 低影响
每个检查项 = 验证 + 修正 + 回归测试
```

### Phase A: 代码语法准确性审查 (最高优先级)

#### A1. 快速参考卡代码验证

**检查对象**: `quick-refs/` 下6个文档中的所有代码片段

**检查方法**:
1. 从每个代码片段中提取CAA宏和API调用
2. 与对应.edu模块中的实际源码交叉验证
3. 标记所有无法在源码中找到匹配的代码

**验证规则**:
```
CAA宏验证:
  - CATDeclareInterface    → 在头文件中搜索确认存在
  - CATImplementInterface  → 在cpp文件中搜索确认存在
  - CATDeclareClass        → 在头文件中搜索确认存在
  - CATImplementClass      → 在cpp文件中搜索确认存在
  - 其他所有CAT*/CAA*宏    → 必须能在源码中找到实例

API调用验证:
  - 每个CATI*接口方法调用 → 必须在某个.h中找到virtual声明
  - 每个函数调用          → 必须在某个.h中找到声明
  - 返回值类型            → HRESULT, void, xxx_var 等
```

**输出**: 每个代码片段标注 ✅验证通过 / ❌语法错误 / ⚠️未验证

#### A2. API接口文档验证

**检查对象**: `api-reference/interfaces/` 下23个接口文档

**检查方法**:
1. 读取接口文档中的方法签名
2. 回到源文件读取原始.h文件
3. 逐方法比对签名是否一致
4. 检查是否遗漏方法

**验证项**:
- [ ] 方法名拼写正确
- [ ] 返回类型正确
- [ ] 参数列表完整
- [ ] const修饰符正确
- [ ] 纯虚声明(=0)标记正确
- [ ] 无遗漏方法

#### A3. 故障排除指南验证

**检查对象**: `troubleshooting/` 下3个文档

**检查方法**:
1. 提取每个错误代码和错误描述
2. 在CAA源码和HTML文档中搜索相关错误
3. 检查解决方案是否引用了真实的CAA API
4. 标记所有虚构的解决方案

**需要验证的典型CAA错误**:
```
编译器错误:
  - C2039: "CATXXX" 不是 "CAYYY" 的成员
  - C2065: 未声明的标识符 (通常缺少 #include)
  - C2504: 基类未定义 (通常缺少 LINK_WITH)

链接器错误:
  - LNK2001/LNK2019: 未解析的外部符号
  - TIE 文件缺失导致
  - ExportedBy 宏问题

运行时错误:
  - E_NOINTERFACE: QI 失败
  - CATNullPointer: 空指针
  - CATIA 启动崩溃: 组件注册失败
```

### Phase B: 内容完整性审查 (高优先级)

#### B2. 接口遗漏补全

**检查对象**: 全部863个候选接口头文件

**检查方法**:
1. 扫描所有 `PublicInterfaces/*.h` 和 `LocalInterfaces/*.h`
2. 用以下规则识别接口:
   - 含 `CATDeclareInterface` → CAA接口
   - 含 `CATDeclareClass` → CAA类
   - 类名以 `CATI` 开头 → 可能的框架接口（非edu示例）
   - 类名以 `CAA` 开头且继承自CATBaseUnknown → 示例接口
3. 对每个识别到的接口生成标准格式文档

**补全规则**:
```python
for header_file in all_headers:
    if "CATDeclareInterface" in content:
        # 这是接口 → 提取方法签名
        extract_interface(header_file)
    elif "CATDeclareClass" in content:
        # 这是类 → 提取公共方法
        extract_class(header_file)
```

**预期结果**: 接口文档从23个增长到200+个

#### B3. TIE文件解析（QI路径提取）

**检查对象**: 所有 `TIE_*.tsrc` 文件

**检查方法**:
1. 查找所有TIE文件
2. 解析TIE文件格式（通常是简单的include列表）
3. 构建接口→实现的映射关系

```
TIE文件格式通常为:
// TIE_CATIMyInterface.tsrc
#include "CATIMyInterface.h"          // 接口定义
#include "MyImplementation.h"         // 实现类
TIE_CAAIxxx(CATIMyInterface, MyImpl); // QI绑定宏
```

### Phase C: 内容准确性修正 (中优先级)

#### C1. Markdown重转换

**检查对象**: Phase B1中评级为C/D的文档

**修正方法**:
1. 编写改进版HTML解析器：
   - 正确处理 `<pre>` 标签 → 转为 ```cpp 围栏
   - 保留 `<b>` / `<i>` 标签内容（加粗/斜体标记）
   - 处理HTML表格 → Markdown表格
   - 保留图片路径引用
   - 保留内部链接（转为相对路径）

2. 增量更新：只重新转换质量不达标的文档

**改进版转换规则**:
```python
def improved_html_to_md(html_content):
    soup = BeautifulSoup(html_content)
    
    # 1. 先提取所有<pre>块，保存为代码片段
    code_blocks = []
    for pre in soup.find_all('pre'):
        code = pre.get_text(#)
        # 清理HTML实体
        code = html.unescape(code)
        code_blocks.append(code)
        pre.replace_with(f"__CODE_BLOCK_{len(code_blocks)-1}__")
    
    # 2. 转换剩余HTML
    md = html2text(html_content)
    
    # 3. 还原代码块
    for i, code in enumerate(code_blocks):
        md = md.replace(
            f"__CODE_BLOCK_{i}__",
            f"```cpp/n{code}/n```"
        )
    
    # 4. 转换表格
    # ... table conversion logic
    
    return md
```

#### C2. 标签去噪

**检查对象**: 全部1149个文档的frontmatter tags

**修正方法**:
1. 加载所有文档的tags列表
2. 用以下规则过滤：
   - 只保留 `CATI*` 开头的接口名
   - 只保留 `CAT*` 开头且长度>8的类名（过滤CATBaseUnknown等通用名）
   - 移除模块名（如CAAApplicationFrame）
   - 移除命令ID（如CAAAfrViewerFeedbackCmdId）
3. 添加缺失的关键标签（如对应的工作台名、功能分类）

**过滤规则**:
```python
KEEP_PREFIXES = ['CATI', 'CATPtr', 'CATList']
REMOVE_PATTERNS = [
    r'^CAA/w+edu$',      # 模块名
    r'^CAA/w+CmdId$',    # 命令ID
    r'^ExportedBy',       # 导出宏
    r'^IID_',             # IID
]
MIN_TAG_LENGTH = 8        # 最短标签长度
```

#### C3. 目录结构优化

**检查对象**: `use-cases/` 目录

**修正方法**:
1. 建立清晰的模块名映射表:

caaafrcases/        → application-frame/
caadlgcases/        → dialog/
caagsicases/        → gsm-interfaces/

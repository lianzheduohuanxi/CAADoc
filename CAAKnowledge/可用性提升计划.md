## 目标

## P0: 核心能力 — 接口方法签名提取

**目标**: 让 agent 能查询任意接口的方法签名、参数和返回值。

**当前状态**: 772 个接口文档全是空壳模板，`knowledge_base.json` 中无方法信息。

**方案**: 从 `.h` 源文件提取 `virtual ... = 0` 方法声明，生成结构化接口文档。

### P0-1: 接口方法提取器

```python
# scripts/extract_interface_methods.py
"""
扫描所有 .edu/PublicInterfaces/*.h 和 .edu/LocalInterfaces/*.h，
提取每个接口/类的:
  - 方法签名 (virtual HRESULT MethodName(params) = 0;)
  - 参数列表 (类型 + 名称)
  - 返回值类型
  - 注释 (方法前的 // 或 /* */ 块)
  - 继承基类
  - include 依赖
  - 类注释 (Abstract of the class)

输出: data/interface_methods.json
"""
```

**输入示例** (`CAAISysAccess.h`):
```cpp
virtual HRESULT SetContainer(CATBaseUnknown * iContainer) =0 ;
virtual HRESULT GetContainer(CATBaseUnknown ** oContainer) =0 ;
```

**输出示例** (`data/interface_methods.json`):
```json
{
  "CAAISysAccess": {
    "name": "CAAISysAccess",
    "base": "CATBaseUnknown",
    "module": "CAASystem",
    "description": "Interface which associates a container with its objects",
    "methods": [
      {
        "name": "SetContainer",
        "return_type": "HRESULT",
        "params": [{"type": "CATBaseUnknown *", "name": "iContainer"}],
        "virtual": true,
        "pure_virtual": true
      },
      {
        "name": "GetContainer",
        "return_type": "HRESULT",
        "params": [{"type": "CATBaseUnknown **", "name": "oContainer"}],
        "virtual": true,
        "pure_virtual": true
      }
    ],
    "includes": ["CATBaseUnknown.h", "CAASysGeoModelInf.h"],
    "source_file": "CAASystem.edu/PublicInterfaces/CAAISysAccess.h"
  }
}
```

## P1: 格式修复 — Use-Case 文档清理

**目标**: 修复 1,374 个 use-case 文档的格式损坏，让代码块可读。

**当前状态**: 83.4% 文档 frontmatter 损坏，代码块被 ` ```vbscript ` 围栏碎片化。

### P1-1: Frontmatter 修复

**问题**: `---\n```vbscript\n` 替代了正确的 `---\n`。

**方案**: 批量扫描所有 use-case .md，修复 frontmatter：

```python
# scripts/fix_frontmatter.py
"""
1. 检测以 ---\n```vbscript 开头的文件
2. 提取真正的 YAML frontmatter 字段 (title, category, module, tags, source_file, converted)
3. 重写为正确的 ---\n...\n--- 格式
4. 修复 source_file 扩展名: .htmmd → .htm
"""
```

### P1-2: 代码块去碎片化

**问题**: ` ```vbscript ` 围栏错误地插入在 C++ 代码行中间。

**方案**: 智能合并碎片化的代码块：

```python
# scripts/defragment_code_blocks.py
"""
1. 识别被 ```vbscript 打断的连续代码行
2. 移除中间的错误围栏标记
3. 将 VBScript 代码包裹为 ```vbscript，C++ 代码包裹为 ```cpp
4. 合并重复的相邻段落
"""
```

**处理逻辑**:
```
检测模式:
  - C++ 关键字 (HRESULT, virtual, class, #include) → ```cpp
  - VBScript 关键字 (Set, Dim, CATIA., .Add() → ```vbscript
  - 连续代码行合并为一个代码块
```

### P1-3: 内容去重

**问题**: 很多文档中同一段文字重复出现 2-3 次。

**方案**: 检测连续重复段落，只保留一份。

### P1 验收标准

- [ ] Frontmatter 正确率 ≥95%（从当前 16.6%）
- [ ] 代码块有正确的语言标记（```cpp 或 ```vbscript）
- [ ] 代码行不再被围栏打断
- [ ] 重复段落已去重

## P2: 增强 — 搜索与交叉引用

**目标**: agent 能通过关键词搜索定位文档，并在接口和用例之间导航。

### P2-1: 搜索索引重建

**当前问题**: search-index.json 索引了大量报告文件（审查报告、执行报告等），而非技术文档。

**方案**:

```python
# scripts/rebuild_search_index.py
"""
1. 只索引以下目录:
   - api-reference/interfaces/
   - use-cases/
   - quick-refs/
   - troubleshooting/
2. 排除所有报告文件 (*报告*.md, *执行*.md, *审查*.md)
3. 索引字段: title, interface_name, method_names, code_keywords, summary
4. 提取代码中的关键 API 调用作为搜索关键词
"""
```

**新索引结构**:
```json
{
  "total_docs": 2100,
  "documents": [
    {
      "title": "CAAISysAccess",
      "type": "interface",
      "path": "api-reference/interfaces/CAAISysAccess.md",
      "methods": ["SetContainer", "GetContainer"],
      "keywords": ["container", "CAAGeometry", "access"],
      "summary": "Interface which associates a container with its objects"
    }
  ],
  "keyword_index": {
    "SetContainer": ["api-reference/interfaces/CAAISysAccess.md"],
    "workbench": ["use-cases/caaafrcases/CAAAfrSampleWorkbench.md", ...]
  }
}
```

### P2-2: 接口↔用例交叉引用增强

**当前状态**: 217 个接口有交叉引用，但覆盖率低。

**方案**: 扫描 use-case 文档，检测代码中出现的接口名（`CATI*`, `CAAI*`），自动建立双向链接。

```python
# scripts/build_cross_references.py
"""
1. 扫描所有 use-case .md，提取代码块中的 CATI*/CAAI* 接口名
2. 在接口文档末尾添加 "## 相关案例" 章节
3. 在 use-case 文档末尾添加 "## 涉及的接口" 章节
"""
```

### P2-3: TIE 绑定补充

**目标**: 从 75 个提升到 ≥200 个 TIE 绑定。

**方案**: 扫描所有 `.cpp` 文件中的 `TIE_XXX` 宏调用。

### P2 验收标准

- [ ] 搜索索引仅含技术文档（无报告文件）
- [ ] ≥500 个接口有交叉引用链接
- [ ] keyword_index 支持反向搜索
- [ ] TIE 绑定 ≥200
